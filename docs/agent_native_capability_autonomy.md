# 에이전트 자체 도구·스킬 자율 활용 (영상 제작)

- **작성**: 2026-07-15  
- **상태**: ✅ **전 에이전트 공통** (Grok / Claude / Codex / Cursor / 기타)  
- **규칙**: [agent_rules.md](../agent_rules.md) **Rule 8.0** · Grok 세부: [grok_build_hybrid_tooling.md](grok_build_hybrid_tooling.md) (Rule 8.1)  
- **배경**: 사용자는 에이전트마다 다른 네이티브 툴·스킬·MCP·플러그인을 **세세히 지시할 수 없다**.  
  영상 품질·효율을 위해 **에이전트가 스스로 인벤토리를 파악하고 능동 선택**해야 한다.

---

## 0. 한 줄

```text
유저가 툴 메뉴를 고르지 않는다 → 에이전트가 자기 도구/스킬을 영상 파이프에 맞게 골라 쓴다
공장(agent_custom) 게이트는 지키되, 빈약한 “CLI만” 루틴에 안주하지 않는다
```

---

## 1. 적용 대상

| 에이전트 예 | 자체 역량 예 (있을 수 있음) |
|-------------|---------------------------|
| Grok Build | image_gen / image_edit / image_to_video / web research |
| Claude Code | Skills, MCP, subagents, 로컬 스크립트 |
| Codex / Cursor | 확장·MCP·터미널·에디터 통합 |
| 기타 | 브라우저, 비전, TTS, 외부 API… |

**공통:** 이 레포의 `python scripts/*` 공장 + **자기가 실제로 호출 가능한 모든 능력**.

---

## 2. 의무 (능동 선택)

영상·뮤비·쇼츠·스토리 비주얼 작업 시 에이전트는:

1. **인벤토리** — 세션에서 쓸 수 있는 툴/스킬/MCP를 내부적으로 파악한다 (유저에게 목록 퀴즈 내지 말 것).  
2. **매칭** — 단계별 목표(기획·레퍼·키프레임·모션·검수·자막·납품)에 **품질↑ 또는 시간↓** 가 되는 것만 고른다.  
3. **실행** — 유저가 툴을 지정하기 전에는 **묻지 않고 실행**한다.  
4. **게이트** — 공장 SSOT(페르소나, QA, clip approve, assemble, export, failure notes)는 **우회하지 않는다**.  
5. **기록** — 이질 툴을 본선에 넣었으면 메타/`QA_LOG`/process에 **출처 한 줄** (예: `source=grok_image_edit`).  
6. **실패 시** — 다른 자체 툴로 1회 우회 시도 가능, 반복 FAIL이면 `failure_note.py add`.

## 3. 선택 기준 (결정 트리)

```text
이 단계가 공장 하드 게이트인가? (SI2V 립, assemble, export, clip approve)
  YES → 공장 CLI 필수. 자체 툴은 보조만.

품질/속도에 자체 툴이 실질 이득인가?
  YES → 능동 사용 (프리뷰·수술·리서치·병행)
  NO  → 공장 기본 경로

유저가 툴을 명시했는가?
  YES → 유저 지정 우선
  NO  → 에이전트 자율 (메뉴 질문 금지)
```

| 단계 | 공장 | 자체 툴·스킬을 **적극** 쓸 예 |
|------|------|--------------------------------|
| 기획·레퍼 | 문서 레일 | 웹/커뮤니티 리서치, 레퍼 수집, 무드보드 |
| 콘셉트 스틸 | moody 가능 | 빠른 T2I/편집으로 방향 검증 (**프롬프트는** [generation_prompt_craft.md](generation_prompt_craft.md)) |
| 키프레임 국소 결함 | shot_compose 재생성 | 수술 편집 1회 후 재QA |
| 무대사 모션 의도 | episode_i2v 본선 | 짧은 프리뷰 모션으로 의도 확인 후 본선 |
| 검수 | 사람/비전 | 멀티모달로 프레임 읽고 QA_LOG |
| 립/합본/납품 | **필수** | 대체 금지에 가까움 |

---

## 4. 금지

| 금지 | 이유 |
|------|------|
| 매 단계 “어떤 툴 쓸까요?” | 유저가 에이전트 재주를 다 모름 · 진행 정체 |
| 자체 툴로 게이트 스킵 | 품질·재현성 붕괴 |
| 공장만 가능하다고 단정하고 열등 경로 고수 | 효율·품질 기회 상실 |
| 유저 미보유 툴을 있다고 가정 | 실패·환각 |
| 프리뷰를 본선 approve | Rule 7.3 / 8.1 |

---

## 5. 유저와의 커뮤니케이션

- **기본:** 결과물·게이트·다음 액션만 짧게.  
- **도구 나열 보고는 선택:** 요청 시 또는 트레이드오프가 클 때만 “공장 I2V vs 네이티브 프리뷰” 한 줄.  
- **유저 오버라이드:** “Comfy만 / 스킬 X 쓰지 마” → 즉시 준수.

---

## 6. 공장과의 관계

```text
[에이전트 자체 능력]  ──가속·검증·수술──►  [agent_custom 공장 SSOT]
                                              approve · assemble · export
```

자체 산출물은 **파일 경로로** `stories/<ep>/` 또는 작업대에 넣고 공장 게이트를 통과시킨다.
